# Protocols

In diesem Ordner werden alle Meeting-Protokolle des Projekts gespeichert. Ziel ist es, eine zentrale und einheitliche Dokumentation aller Besprechungen zu gewährleisten.

## Namenskonvention

Jedes Protokoll sollte nach folgendem Muster benannt werden:
```
YYYY-MM-DD_BESPRECHUNGSTHEMA.md
```

**Beispiel**
```
2025-10-14_FeaturePlanning.md
```

## Aufbau eines Protokolls (Kann noch angepasst werden)

### Meeting-Protokoll: (Thema)
**Datum:** YYYY-MM-DD  
**Uhrzeit:** HH:MM – HH:MM  
**Ort / Plattform:** z. B. HTW, Teams / Discord 
**Teilnehmer:** Name 1, Name 2, Name 3  

### Agenda
1. Punkt 1
2. Punkt 2
3. Punkt 3

### Besprochene Themen / Notizen
- Thema 1: Kurze Beschreibung, Entscheidungen, To-Dos
- Thema 2: ...

### Entscheidungen / Beschlüsse
- Entscheidung 1
- Entscheidung 2

### To-Dos / Aufgaben
- [ ] Aufgabe 1 (Zuständig: Name, Frist: YYYY-MM-DD)
- [ ] Aufgabe 2 (Zuständig: Name, Frist: YYYY-MM-DD)

### Nächstes Meeting
**Datum & Uhrzeit:** YYYY-MM-DD, HH:MM  
**Ort / Plattform:** z. B. Discord
